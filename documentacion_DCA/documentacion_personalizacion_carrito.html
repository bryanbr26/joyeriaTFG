Integración de Personalización con Carrito y Pedidos

Documentación técnica — Proyecto Joyería TFG
Fecha: 14 de Abril de 2026

1. Descripción general

Se ha integrado la herramienta de personalización de joyas (canvas de dibujo) con el sistema de carrito y pedidos, permitiendo que un usuario:

  1. Seleccione un producto desde su página de detalle.
  2. Acceda a la herramienta de personalización vinculada a ese producto.
  3. Dibuje, escriba texto o suba imágenes en el canvas como diseño de grabado.
  4. Añada el producto personalizado al carrito con la imagen del grabado asociada.
  5. Al pasar por caja, el grabado se conserva en el historial de pedidos.

El sistema diferencia productos personalizados de los normales: el mismo producto puede estar en el carrito con y sin grabado como líneas separadas, y cada grabado es único (nunca se agrupan).

│
2. Pulsa "Personalizar joya" → abre personaliza.blade.php con contexto del producto

3. Dibuja/escribe/sube imagen en el canvas HTML5
│
4. Pulsa "Añadir a la cesta"
│ ├── @auth → AJAX POST con canvas.toDataURL() (base64)
│ └── @guest → Modal de login
│
5. Servidor: guarda PNG en storage/app/public/grabados/
│ crea entrada en CARRITO con ruta_grabado_personalizado
│
6. En la cesta: aparece con badge "Personalizado" + miniatura del grabado
│
7. Al pasar por caja: se copia ruta_grabado_personalizado a DETALLES_PEDIDO
│
8. En el historial de pedidos: se muestra el grabado junto al producto

5. Rutas añadidas

// Ruta genérica (ya existía)
Route::get('/personaliza-tus-joyas', [PersonalizaController::class, 'personaliza'])
    ->name('personaliza');

// NUEVAS
Route::post('/personaliza-tus-joyas/guardar',
    [PersonalizaController::class, 'guardarGrabado'])
    ->name('personaliza.guardar');

Route::get('/personaliza-tus-joyas/{producto}',
    [PersonalizaController::class, 'personalizaProducto'])
    ->name('personaliza.producto');

Nota: La ruta POST de guardar va antes de la ruta GET con {producto} para evitar que Laravel interprete "guardar" como un ID de producto.

6. PersonalizaController

6.1 Método personalizaProducto()

Recibe un Producto por inyección de modelo y lo pasa a la vista para activar el modo producto:

public function personalizaProducto(Producto $producto)
{
    return view('pages.personaliza', compact('producto'));
}

6.2 Método guardarGrabado()

Endpoint AJAX que recibe la imagen del canvas en base64 y la guarda en storage:

  • Valida autenticación (devuelve 401 JSON si no está logueado).
  • Valida imagen_base64 (string requerido) y producto_id (existe en tabla PRODUCTO).
  • Decodifica el base64, elimina el prefijo data:image/png;base64,.
  • Guarda el archivo PNG en storage/app/public/grabados/ con nombre único: {userId}_{productoId}_{timestamp}.png.
  • Crea una entrada en CARRITO con ruta_grabado_personalizado apuntando al archivo.
  • Los productos personalizados siempre son entradas nuevas (nunca se agrupan con otros del mismo producto).
// Nombre único: evita colisiones
$nombreArchivo = 'grabados/' . Auth::id() . '_' . $producto->id . '_' . time() . '.png';
Storage::disk('public')->put($nombreArchivo, $imagenBinaria);

// Siempre es una entry nueva (cada grabado es único)
Carrito::create([
    'id_usuario' => $userId,
    'id_producto' => $producto->id,
    'cantidad' => 1,
    'ruta_grabado_personalizado' => $nombreArchivo
]);

7. Migraciones de base de datos

7.1 Columna en CARRITO

Almacena la ruta del grabado mientras el producto está en el carrito (dato temporal, se elimina al vaciar el carrito):

$table->string('ruta_grabado_personalizado', 300)->nullable()->after('id_producto');

7.2 Columna en DETALLES_PEDIDO

Almacena la ruta del grabado de forma permanente en el historial de pedidos. Se copia desde CARRITO durante el checkout:

$table->string('ruta_grabado_personalizado', 300)->nullable()->after('id_producto');

7.3 Ciclo de vida del dato

Personalización → canvas.toDataURL()
↓ AJAX POST (base64)
Storage → storage/app/public/grabados/{user}_{prod}_{time}.png
↓ ruta guardada
CARRITO → ruta_grabado_personalizado (temporal)
↓ checkout()
DETALLES_PEDIDO → ruta_grabado_personalizado (permanente)
↓
CARRITO se vacía (la imagen sigue en storage y en DETALLES_PEDIDO)

¿Por qué en ambas tablas?

  • CARRITO necesita el campo para mostrar el grabado en la cesta antes de comprar y para diferenciar líneas del mismo producto con grabados distintos.
  • DETALLES_PEDIDO necesita el campo porque el carrito se vacía al pagar, y la imagen debe persistir en el historial.

8. Diferenciación de productos normales vs personalizados

Un producto normal (sin personalización) tiene ruta_grabado_personalizado = NULL. Los productos normales se agrupan en el carrito (si el usuario añade el mismo producto dos veces, se incrementa la cantidad).

Un producto personalizado tiene ruta_grabado_personalizado con la ruta al archivo PNG. Cada grabado genera una línea separada en el carrito, porque cada diseño es único.

Escenario Entradas en CARRITO
Anillo sin personalizar × 3 1 fila (cantidad = 3, grabado = NULL)
Anillo con grabado "Ana" 1 fila (cantidad = 1, grabado = grabados/1_5_001.png)
Anillo con grabado "Pedro" 1 fila (cantidad = 1, grabado = grabados/1_5_002.png)
Total 3 filas separadas del mismo producto

  • Badge: Etiqueta oscura "Personalizado" con icono de pincel junto al nombre.
  • Miniatura: Imagen del grabado de 50px de alto. Click abre la imagen completa en nueva pestaña.

10. Visualización en el historial de pedidos

En pedidos/show.blade.php, los detalles del pedido que tienen grabado personalizado muestran la misma información:

  • Badge "Personalizado" junto al nombre del producto.
  • Miniatura del grabado con enlace para verlo en tamaño completo.

Esto permite al usuario (y a los administradores) identificar qué productos fueron personalizados y ver exactamente qué diseño se pidió.

11. Modificaciones en el checkout

En CarritoController::checkout(), al crear cada DetallePedido, se copia el campo ruta_grabado_personalizado desde el item del carrito:

DetallePedido::create([
    'cantidad' => $item->cantidad,
    'precio_unitario' => $item->producto->precio,
    'id_pedido' => $pedido->id,
    'id_producto' => $item->producto->id,
    'ruta_grabado_personalizado' => $item->ruta_grabado_personalizado  // ← NUEVO
]);

Si el item no es personalizado, el valor será NULL y el pedido se comporta como antes.

13. Resumen de tecnologías utilizadas

Tecnología Uso
HTML5 Canvas Herramienta de dibujo con lápiz, texto e imágenes para el diseño del grabado.
canvas.toDataURL() Exporta el contenido del canvas como imagen PNG en base64 para enviar al servidor.
Fetch API (AJAX POST) Envía la imagen del grabado al servidor sin recargar la página.
Laravel Storage Storage::disk('public')->put() para guardar las imágenes de grabado en el servidor.
base64_decode (PHP) Decodifica la imagen base64 recibida del canvas para almacenarla como archivo binario PNG.
Inyección de modelo Producto $producto en la ruta para cargar automáticamente el producto.
Directivas Blade @auth/@else Renderiza el botón AJAX para autenticados o el modal de login para invitados.
Bootstrap 5 Badges, modales, botones y layout consistente con el resto del sitio.
