Mejoras del Carrito y Sistema de Pedidos

Documentación técnica — Proyecto Joyería TFG
Fecha: 14 de Abril de 2026

1. Descripción general

Se han implementado mejoras significativas en el sistema de carrito y un sistema completo de pedidos. Los cambios incluyen:

  • Control de cantidad en la cesta con botones +/− que respetan el stock disponible, actualizando precios en tiempo real vía AJAX.
  • Proceso de checkout funcional: al "Pasar por caja" se crea un pedido en la base de datos, se descuenta el stock de cada producto y se vacía el carrito.
  • Historial de pedidos con listado de todos los pedidos del usuario y vista de detalle por pedido.
  • Nueva migración para añadir id_producto a la tabla DETALLES_PEDIDO.

4. Mejoras en el Carrito

4.1 Control de cantidad con validación de stock

Se reemplazó el badge estático de cantidad por controles interactivos −/+ que actualizan vía AJAX:

┌────────────────────────────────────────────────────────┬──────┐
│ [Imagen] Nombre del producto 200.00€ │
│ Descripción abreviada... 100.00€/ud │
│ │
│ [Eliminar] Cantidad [−] 2 [+] (5 disp.) │
└────────────────────────────────────────────────────────┴──────┘

Comportamiento de los botones:

  • El botón − se deshabilita cuando la cantidad es 1 (mínimo).
  • El botón + se deshabilita cuando la cantidad alcanza el stock disponible del producto.
  • Cada click envía una petición PATCH vía Fetch API al endpoint /carrito/{id}/cantidad.
  • La respuesta JSON actualiza en tiempo real: cantidad, subtotal del item, precio total de la cesta y contador de items.

4.2 Método actualizarCantidad()

public function actualizarCantidad(Request $request, $id)
{
    $item = Carrito::with('producto')
                   ->where('id', $id)
                   ->where('id_usuario', Auth::id())
                   ->first();

    $nuevaCantidad = (int) $request->input('cantidad', 1);

    // Limitar entre 1 y stock disponible
    if ($nuevaCantidad < 1) $nuevaCantidad = 1;
    if ($nuevaCantidad > $item->producto->stock) $nuevaCantidad = $item->producto->stock;

    $item->cantidad = $nuevaCantidad;
    $item->save();

    // Recalcular y devolver totales
    return response()->json([
        'success' => true,
        'cantidad' => $nuevaCantidad,
        'subtotal' => ...,
        'totalPrice' => ...,
        'totalItems' => ...
    ]);
}

4.3 Validación de stock en agregar()

Se añadió validación al método de añadir al carrito desde la vista de detalle:

  • Si el producto ya está en carrito y cantidad >= stock, devuelve error JSON.
  • Si el producto tiene stock <= 0, no se crea la entrada.

5. Proceso de Checkout

5.1 Flujo completo

Usuario pulsa "Pasar por caja"
│
├── 1. Confirmar compra (JS confirm())
│
├── 2. Verificar stock de todos los items
│ └── Si alguno excede stock → error, volver atrás
│
├── 3. DB::beginTransaction()
│ ├── Calcular total
│ ├── Crear registro en PEDIDO
│ ├── Por cada item del carrito:
│ │ ├── Crear DETALLES_PEDIDO (cantidad, precio, id_producto)
│ │ └── Descontar stock del PRODUCTO
│ └── Eliminar todos los items del CARRITO del usuario
│
├── 4. DB::commit()
│
└── 5. Redirigir a historial de pedidos con mensaje de éxito

5.2 Integridad transaccional

Todo el proceso de checkout se ejecuta dentro de una transacción de base de datos (DB::beginTransaction()). Si ocurre cualquier error, se ejecuta DB::rollBack() y ningún dato es modificado. Esto garantiza que:

  • No se creen pedidos sin descontar stock.
  • No se descuente stock sin crear el pedido.
  • No se vacíe el carrito si la creación del pedido falla.

7. Migración de base de datos

7.1 Nueva columna id_producto en DETALLES_PEDIDO

Se creó una migración para añadir la columna id_producto a la tabla DETALLES_PEDIDO, permitiendo vincular directamente qué producto fue comprado en cada detalle de pedido:

Schema::table('DETALLES_PEDIDO', function (Blueprint $table) {
    $table->unsignedInteger('id_producto')->nullable()
          ->index('dp_id_producto')
          ->after('id_pedido');
    $table->foreign(['id_producto'], 'DETALLES_PEDIDO_ibfk_2')
          ->references(['id'])->on('PRODUCTO')
          ->onUpdate('CASCADE')
          ->onDelete('SET NULL');
});

Importante: Es necesario ejecutar php artisan migrate para aplicar esta migración.

7.2 Cambio en el modelo DetallePedido

Se añadió id_producto al array $fillable y se cambió la relación producto():

// Antes: hasOne inverso a través del campo del producto
public function producto() {
    return $this->hasOne(Producto::class, 'id_detalles_pedido');
}

// Después: belongsTo directo con la nueva FK
public function producto() {
    return $this->belongsTo(Producto::class, 'id_producto');
}
