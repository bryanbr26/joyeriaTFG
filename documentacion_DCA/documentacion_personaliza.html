Personalizador de Joyas

Documentación técnica — Proyecto Joyería TFG
Fecha: 5 de Abril de 2026

1. Descripción general

Se ha implementado una herramienta interactiva de personalización de joyas dentro de la página /personaliza-tus-joyas del proyecto Laravel. La herramienta permite al usuario dibujar libremente, escribir texto y subir imágenes sobre un canvas rectangular blanco, simulando un boceto de diseño de joya.

La funcionalidad está inspirada en herramientas como el personalizador de Pandora, pero simplificada a tres acciones principales: dibujar, escribir y cargar imagen.

3. Arquitectura del canvas

Se utilizan dos canvas HTML5 superpuestos para evitar que el cursor de texto se quede grabado en el dibujo:

┌──────────────────────────────────┐
│ canvas-cursor (overlay transparente) │ ← Solo el cursor "|" parpadeante
│ pointer-events: none │
├──────────────────────────────────┤
│ canvas-joya (canvas principal) │ ← Dibujos, texto e imágenes
│ border: 1px solid #ccc │
└──────────────────────────────────┘
  • canvas-joya (900×450px): canvas principal donde se dibujan los trazos del lápiz, el texto confirmado y las imágenes cargadas.
  • canvas-cursor (900×450px): canvas transparente posicionado encima con position: absolute y pointer-events: none. Solo se usa para mostrar el cursor parpadeante "|" del modo texto. Al estar en un canvas separado, el cursor nunca se "quema" en el dibujo principal.

4. Herramientas implementadas

4.1 Lápiz (dibujo libre)

Permite dibujar trazos libres sobre el canvas. Funciona capturando los eventos mousedown, mousemove y mouseup del canvas:

  • mousedown: marca el inicio del trazo y guarda la posición inicial.
  • mousemove: mientras el ratón está pulsado, dibuja líneas cortas (ctx.lineTo) desde la última posición hasta la actual, creando un trazo continuo.
  • mouseup: finaliza el trazo y guarda el estado en la pila de deshacer.

También se gestionan mouseleave y mouseenter: al salir del canvas se pausa el dibujo, y al volver a entrar con el botón pulsado se retoma desde la nueva posición (sin dibujar una línea recta desde donde salió).

El color se configura con un <input type="color"> y el grosor con un <select>.

4.2 Texto en vivo

Al activar el modo texto y hacer clic en el canvas, el usuario puede escribir directamente con el teclado y ver el texto aparecer en tiempo real:

  1. Se guarda el estado actual del canvas como ImageData (antes de escribir).
  2. Cada vez que se pulsa una tecla, se restaura ese estado guardado y se redibuja todo el texto encima → así cada carácter nuevo aparece sin acumular artefactos.
  3. El cursor "|" se dibuja en el canvas overlay con un setInterval que alterna su visibilidad cada 500ms.
  4. Enter → salto de línea (se añade una nueva línea al array lineasTexto).
  5. Escape o clic en otro sitio → confirma el texto, detiene el parpadeo y guarda el estado. 
// Estructura de datos del texto multilínea:
let lineasTexto = ['Primera línea', 'Segunda línea', ''];
//                                                   ↑ línea actual donde se escribe

4.3 Subir imagen

Un <input type="file"> oculto permite seleccionar una imagen del ordenador. La imagen se carga con la API FileReader / URL.createObjectURL y se dibuja en el canvas centrada y escalada proporcionalmente para que quepa sin deformarse.

const escala = Math.min(canvas.width / img.width, canvas.height / img.height, 1);
// Se usa Math.min con 1 para que imágenes pequeñas no se agranden

5. Sistema de deshacer

Se mantiene una pila de estados (pilaDeshacer) que almacena capturas del canvas como objetos ImageData:

  • Guardar estado: después de cada acción (trazo dibujado, texto confirmado, imagen subida, limpiar), se llama a ctx.getImageData() y se añade a la pila.
  • Deshacer: se elimina el último estado y se restaura el anterior con ctx.putImageData().
  • Límite: máximo 30 estados para no consumir demasiada memoria.

Importante: se usa ImageData (síncrono) en lugar de toDataURL + new Image() (asíncrono). Esto evita race conditions donde operaciones asíncronas interferían entre sí y causaban que el cursor del texto se quedara grabado en el canvas.

6. Gestión de eventos táctiles

Para que la herramienta funcione en dispositivos táctiles (tablets, móviles), se duplican los eventos de ratón con sus equivalentes táctiles:

Ratón Táctil
mousedown touchstart
mousemove touchmove
mouseup touchend

Se usa e.preventDefault() en los eventos táctiles para evitar el scroll de la página mientras se dibuja, y { passive: false } en el listener.

7. Funcionalidad de guardar

El botón "Guardar" exporta el contenido del canvas como imagen PNG descargable:

const enlace = document.createElement('a');
enlace.download = 'mi-joya-personalizada.png';
enlace.href = canvas.toDataURL('image/png');
enlace.click();

Se crea un enlace temporal con el atributo download y se simula un clic para iniciar la descarga automáticamente.
