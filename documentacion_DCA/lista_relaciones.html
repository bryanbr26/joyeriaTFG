Relaciones de base de datos

Origen Destino Cardinalidad Acciones Uso funcional
DIRECCIONES_ENVIO.id_usuario USUARIO.id Un usuario tiene muchas direcciones. Update cascade, delete cascade. Guardar direcciones disponibles para pedidos.
PEDIDO.id_usuario USUARIO.id Un usuario realiza muchos pedidos. Update cascade, delete cascade. Historial de compras por cliente.
PEDIDO.id_direcciones_envio DIRECCIONES_ENVIO.id Una direccion puede aparecer en muchos pedidos. Update cascade, delete set null. Direccion usada en el envio del pedido.
PEDIDO.id_metodos_pago METODOS_PAGO.id Un metodo de pago se usa en muchos pedidos. Update cascade, delete set null. Clasificar la forma de pago elegida.
DETALLES_PEDIDO.id_pedido PEDIDO.id Un pedido contiene muchas lineas. Update cascade, delete cascade. Productos, cantidades y precios del pedido.
DETALLES_PEDIDO.id_producto PRODUCTO.id Un producto puede aparecer en muchas lineas de pedido. Update cascade, delete set null. Relacionar cada linea con la joya comprada.
PAGOS_REDSYS.id_pedido PEDIDO.id Un pedido tiene pago Redsys asociado. Update cascade, delete cascade. Guardar autorizacion, estado y respuesta de Redsys.
CARRITO.id_usuario USUARIO.id Un usuario tiene muchos items de carrito. Update cascade, delete cascade. Cesta activa del cliente.
CARRITO.id_producto PRODUCTO.id Un producto puede estar en muchos carritos. Update cascade, delete cascade. Producto seleccionado antes de comprar.
FAVORITOS.id_usuario USUARIO.id Un usuario tiene muchos favoritos. Update cascade, delete cascade. Lista de deseos del cliente.
FAVORITOS.id_producto PRODUCTO.id Un producto puede estar en muchos favoritos. Update cascade, delete cascade. Producto guardado como favorito.
IMAGENES_PRODUCTO.id_producto PRODUCTO.id Un producto tiene muchas imagenes. Update cascade, delete cascade. Galeria e imagen principal del producto.
LOGS_USUARIOS.id_usuario USUARIO.id Un usuario genera muchos logs. Update cascade, delete cascade. Registro de acciones de usuario.
PRODUCTO.id_detalles_pedido DETALLES_PEDIDO.id Relacion opcional heredada. Update cascade, delete set null. Campo existente en migraciones; no es la relacion principal de venta.
En el modelo Pedido, pagoRedsys() se declara como hasOne. En base de datos PAGOS_REDSYS.id_pedido no tiene restriccion unique, por lo que la intencion de la aplicacion es 1:1 aunque la base de datos permitiria mas de un registro si se insertan manualmente.
