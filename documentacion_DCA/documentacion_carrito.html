Sistema de Carrito de Compras

Documentación técnica — Proyecto Joyería TFG
Fecha: 7 de Abril de 2026

1. Descripción general

Se ha implementado un sistema de carrito de compras funcional. Incluye la lógica necesaria para que tanto la vista de detalle del artículo permita agregar productos de manera correcta (impidiendo su acceso si no está logeado), como una vista propia del carrito donde se muestran todos los productos subidos, así como el cálculo del precio y la opción de eliminación.

2. Archivos principales involucrados

Archivo Función
app/Http/Controllers/CarritoController.php Controlador nuevo con métodos agregar(), index() y eliminar().
routes/web.php Nuevas rutas para la cesta protegidas por inicio de sesión (algunas explícitas, otras bajo el grupo de auth).

3. Rutas y Controlador

3.1 Rutas de Carrito (web.php)

Se ha creado un grupo de rutas protegidas obligatoriamente bajo uso del middleware de autenticación (auth):

Route::middleware('auth')->group(function () {
    Route::get('/carrito', [CarritoController::class, 'index'])->name('carrito.index');
    Route::post('/carrito/agregar/{producto}', [CarritoController::class, 'agregar'])->name('carrito.agregar');
    Route::delete('/carrito/{id}', [CarritoController::class, 'eliminar'])->name('carrito.eliminar');
});

3.2 CarritoController

Responsable de las tres acciones principales:

  • agregar(): Funcionalidad tipo AJAX, comprueba si un producto está presente de este usuario para sumar la cantidad o crear la entrada.
  • index(): Función central para la vista. Recolecta las relaciones con Producto::with('producto.imagenes') de las entradas y calcula a código el gasto total.
  • eliminar(): Procedimiento típico de eliminación validando coincidencia con `id_usuario`.

4. Modificaciones en Detalle de Producto

Se modificó resources/views/joyas/show.blade.php de esta manera:

Directiva @auth - Verifica el cliente
│
├── [SI] Logeado
│ └── Ejecuta script AJAX (Fetch), envía Token CSRF, muestra animación de carga, y alert de éxito. Aumenta carrito de BBDD.
│
└── [NO] Invitado
└── Llama a Modal de Bootstrap con botones redirigiendo a login & register.

Código del script fetch en JS vanilla para que no se recargue toda la página para esto:

fetch(url, {
    method: 'POST',
    headers: {
        'X-CSRF-TOKEN': token,
        'Accept': 'application/json',
        'Content-Type': 'application/json'
    }
})

5. Vista General de la Cesta

5.1 Estructura Estilística

La vista completa carrito/index.blade.php tiene un enfoque muy elegante. Siguiendo en exclusiva las utilidades de Bootstrap 5 y ligeros retoques por CSS dentro de la misma vista, la estructura cuenta con:

  • Color de fondo de página tenue #f8f9fa.
  • Bordeado gris que define cada caja de producto listada usando flexbox.
  • Soporta responsividad: en móvil es colapsable verticalmente para la imagen y la información.
  • Uso de botón javascript:history.back() en la zona superior.
┌────────────────────────────────────────────────────────┬─────┐
│ [Imagen] Lorem ipsum dolor sit amet 100€ │
│ La marca o descripcion abreviada... │
│ │
│ [Eliminar producto] Cantidad [ 1 ] │
└────────────────────────────────────────────────────────┴─────┘

5.2 Elementos Internos de la Vista

  • Boton de Eliminar: Se trata de un form encapsulado con método DELETE hacia la ruta de acción.
  • Badge de Cantidad: Simulación limpia y minimalista, en un fondo ligeramente más oscuro con bordes sutiles.
  • Placeholder Empty: Cesta prefabricada alternativa por si se abre sin items para sugerir ir al inicio (Seguir Comprando).
  • Pasar de Caja final: Bloque con un botón Alert demostrativo indicando las partes de pago pendientes.

6. Resumen de tecnologías utilizadas

Tecnología Uso
Componente Modal Bstrp Se encarga de la pausa a invitados de agregar elementos forzando el paso por caja.
Vanilla Fetch JS Añade de forma asíncrona un artículo al carrito desde show.blade.php
Eloquent Relacional Vincula belongsTo(User::class) y belongsTo(Producto::class) dentro del modelo.
