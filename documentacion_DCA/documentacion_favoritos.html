Sistema de Favoritos

Documentación técnica — Proyecto Joyería TFG
Fecha: 12 de Abril de 2026

1. Descripción general

Se ha implementado un sistema de favoritos completo que permite a los usuarios autenticados guardar productos como favoritos desde la vista de detalle, y gestionar su lista de favoritos en una página dedicada. Los usuarios no autenticados son redirigidos a iniciar sesión o registrarse antes de poder usar esta funcionalidad.

El sistema incluye:

  • Botón de corazón en la vista de detalle del producto con toggle AJAX (añadir/eliminar favorito).
  • Página de favoritos con listado completo, opción de eliminar de favoritos y añadir al carrito.
  • Modal de autenticación para usuarios invitados que intentan marcar favoritos.
  • Enlace en el header del sitio hacia la página de favoritos.

3. Rutas y Controlador

3.1 Rutas de Favoritos (web.php)

La ruta de toggle queda fuera del middleware auth para poder devolver un JSON 401 en las peticiones AJAX cuando el usuario no está autenticado. El resto de rutas sí están protegidas:

// Toggle va fuera del middleware auth para devolver JSON 401 en AJAX
Route::post('/favoritos/toggle/{producto}', [FavoritoController::class, 'toggle'])
    ->name('favoritos.toggle');

Route::middleware('auth')->group(function () {
    Route::get('/favoritos', [FavoritoController::class, 'index'])
        ->name('favoritos.index');
    Route::delete('/favoritos/{id}', [FavoritoController::class, 'eliminar'])
        ->name('favoritos.eliminar');
    Route::post('/favoritos/{id}/carrito', [FavoritoController::class, 'agregarAlCarrito'])
        ->name('favoritos.agregarCarrito');
});

3.2 FavoritoController

Responsable de las cuatro acciones principales:

  • index(): Carga los favoritos del usuario autenticado con sus relaciones de producto e imágenes y los pasa a la vista.
  • toggle(): Recibe peticiones AJAX POST. Si el producto ya es favorito lo elimina; si no lo es, lo crea. Devuelve JSON con el estado resultante (favorito: true/false).
  • eliminar(): Elimina un favorito por su ID validando que pertenezca al usuario actual.
  • agregarAlCarrito(): Toma un favorito, localiza el producto asociado y lo añade al carrito (incrementando cantidad si ya existe).

4. Modificaciones en Detalle de Producto

Se modificó resources/views/joyas/show.blade.php para hacer funcional el botón de favoritos:

4.1 Detección del estado de favorito

Al cargar la página, se consulta si el producto ya es favorito del usuario actual mediante un bloque @php:

@php
    $esFavorito = false;
    if(Auth::check()) {
        $esFavorito = \App\Models\Favorito::where('id_usuario', Auth::id())
                        ->where('id_producto', $producto->id)
                        ->exists();
    }
@endphp

4.2 Lógica del botón corazón

Directiva @auth - Verifica el cliente
│
├── [SI] Logeado
│ ├── Muestra corazón relleno rojo si ya es favorito
│ ├── Muestra corazón vacío si no lo es
│ └── Click → Fetch AJAX POST a /favoritos/toggle/{id} → Toggle icono
│
└── [NO] Invitado
└── Click → Abre Modal Bootstrap con botones a login & register

4.3 Modal de autenticación unificado

Se actualizó el modal existente para cubrir tanto el carrito como favoritos. El texto ahora indica:

"Para añadir productos a la cesta o a favoritos debes iniciar sesión
o crear una cuenta nueva."

El icono se cambió a bi-person-lock para reflejar mejor el contexto de autenticación.

5. Vista de Favoritos

5.1 Estructura Visual

La vista favoritos/index.blade.php sigue exactamente el mismo patrón de diseño que la cesta, reutilizando las clases CSS item-box, item-image e item-details:

┌────────────────────────────────────────────────────────┬──────┐
│ [Imagen] Nombre del producto 100.00€ │
│ Marca · Descripción abreviada... │
│ │
│ [♥ Eliminar] [🛒 Añadir a cesta] │
└────────────────────────────────────────────────────────┴──────┘

5.2 Elementos de cada item

  • Imagen: Cuadro de 150×150px con la imagen del producto o icono placeholder de gema.
  • Información: Nombre, marca, descripción abreviada (70 chars) y precio con formato.
  • Eliminar de favoritos: Formulario con método DELETE y confirmación onsubmit.
  • Añadir a la cesta: Formulario POST que envía el producto al carrito. Se deshabilita si stock <= 0.

5.3 Estado vacío

Si el usuario no tiene favoritos, se muestra un bloque con borde discontinuo, icono de corazón, mensaje descriptivo y botón Explorar joyas que redirige al inicio.
