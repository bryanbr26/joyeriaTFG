Página de Detalle de Producto

Documentación técnica — Proyecto Joyería TFG
Fecha: 7 de Abril de 2026

1. Descripción general

Se ha implementado una página de detalle de producto que se muestra al pulsar sobre cualquier producto del listado de joyas (/collares, /anillos, /pulseras, /pendientes). La página presenta toda la información del producto en un diseño de dos columnas: imagen a la izquierda e información con botones de acción a la derecha.

La página incluye tres botones de acción: Añadir a la cesta, Favoritos (toggle visual) y Personalizar joya (enlace a la herramienta de personalización). Se utilizan exclusivamente clases de Bootstrap 5, sin estilos CSS adicionales.

4. Ruta y controlador

4.1 Definición de la ruta

La ruta está definida dentro del grupo de rutas de joyas por categoría en web.php. Ya existía antes de esta implementación:

Route::prefix('{categoria}')
    ->where(['categoria' => 'collares|anillos|pulseras|pendientes'])
    ->name('joyas.')
    ->group(function () {
        // ... otras rutas ...
        Route::get('/{producto}', [JoyasController::class, 'show'])->name('show');
    });

Gracias al route model binding de Laravel, el parámetro {producto} se resuelve automáticamente al modelo Producto correspondiente por su id.

4.2 Método show() del controlador

El método show() en JoyasController recibe la categoría y el producto, y pasa las variables necesarias a la vista:

public function show($categoria, Producto $producto)
{
    $titulo = $producto->nombre;

    return view('joyas.show', compact('producto', 'categoria', 'titulo'));
}

5. Componentes de la vista

5.1 Breadcrumb de navegación

Se utiliza el componente breadcrumb de Bootstrap para permitir al usuario volver al inicio o al listado de la categoría:

  • Inicio → enlaza a route('index')
  • Categoría (ej: Collares) → enlaza a route('joyas.index', $categoria)
  • Nombre del producto → elemento activo (página actual)

5.2 Columna izquierda — Imagen del producto

Muestra la imagen del producto ocupando todo el ancho de la columna (col-lg-6). Se usa una directiva Blade condicional para verificar si el archivo de imagen existe en disco:

@if($producto->ruta_grabado && file_exists(public_path('storage/' . $producto->ruta_grabado)))
    <img src="{{ asset('storage/' . $producto->ruta_grabado) }}" ...>
@else
    <div class="bg-light ...">
        <i class="bi bi-gem"></i>  {{-- Icono placeholder --}}
    </div>
@endif

Si el producto no tiene imagen asignada (ruta_grabado vacío) o el archivo no existe físicamente en la carpeta storage, se muestra un placeholder con un icono de gema (bi-gem) de Bootstrap Icons sobre fondo gris claro.

5.3 Columna derecha — Información del producto

Presenta los datos del producto en el siguiente orden:

Elemento Componente Bootstrap Datos mostrados
Nombre <h2 class="fw-bold"> $producto->nombre
Marca <p class="text-muted"> $producto->marca
Precio <h3> number_format($producto->precio, 2) €
Descripción <p> $producto->descripcion (texto completo)
Detalles table table-sm table-borderless Material, color, género, talla, peso, disponibilidad

La tabla de detalles usa directivas @if para mostrar solo los campos que tienen valor, evitando filas vacías. La fila de disponibilidad se muestra siempre: con icono verde (bi-check-circle) y texto "En stock" si hay stock, o icono rojo (bi-x-circle) y texto "Agotado" si el stock es 0.

6. Botones de acción

Los tres botones se agrupan en la parte inferior de la columna derecha. Los dos primeros comparten una fila con d-flex gap-3:

6.1 Añadir a la cesta

Botón principal con clase btn-dark btn-lg que ocupa todo el espacio disponible (flex-grow-1). Se deshabilita automáticamente si el producto no tiene stock:

<button class="btn btn-dark btn-lg flex-grow-1" id="btnAnadirCesta"
        {{ $producto->stock <= 0 ? 'disabled' : '' }}>
    <i class="bi bi-cart-plus me-2"></i>Añadir a la cesta
</button>

Al pulsar, muestra un alert('Producto añadido a la cesta.'). La lógica completa del carrito de compras está pendiente de implementar.

6.2 Botón de favoritos

Botón con icono de corazón (bi-heart) que permite al usuario marcar o desmarcar el producto como favorito. Funciona como un toggle visual mediante JavaScript del lado del cliente:

btnFav.addEventListener('click', function() {
    const icon = this.querySelector('i');
    if (icon.classList.contains('bi-heart')) {
        icon.classList.remove('bi-heart');
        icon.classList.add('bi-heart-fill', 'text-danger');
        alert('Añadido a favoritos.');
    } else {
        icon.classList.remove('bi-heart-fill', 'text-danger');
        icon.classList.add('bi-heart');
        alert('Eliminado de favoritos.');
    }
});
  • Al pulsar, alterna entre bi-heart (corazón vacío) y bi-heart-fill text-danger (corazón relleno rojo).
  • Muestra un alert() indicando la acción realizada.
  • El estado no se persiste en la base de datos — es solo visual. Está pendiente conectarlo con el endpoint toggleFavorito del controlador y requerir autenticación.

6.3 Personalizar joya

Enlace con estilo de botón (btn-outline-secondary w-100) que redirige a la página del personalizador de joyas:

<a href="{{ route('personaliza') }}" class="btn btn-outline-secondary w-100">
    <i class="bi bi-brush me-2"></i>Personalizar joya
</a>

Ocupa todo el ancho de la columna (w-100) y se sitúa debajo de los otros dos botones.

7. Enlace desde el listado

En index.blade.php, cada tarjeta de producto ya estaba envuelta en un enlace que apunta a la ruta joyas.show. No se ha modificado:

<a href="{{ route('joyas.show', [$categoria, $producto]) }}"
   class="text-decoration-none text-dark">
    <div class="card h-100">
        ...
    </div>
</a>

Se usa text-decoration-none text-dark para que el enlace no altere la apariencia del texto dentro de la tarjeta.

8. Variables pasadas a la vista

El controlador pasa las siguientes variables a la vista show.blade.php mediante compact():

Variable Tipo Descripción
$producto Producto (Eloquent) Instancia del producto con todos sus campos (nombre, marca, precio, descripcion, etc.)
$categoria string Slug de la categoría en plural (collares, anillos, pulseras, pendientes)
$titulo string Nombre del producto, usado en el <title> de la página vía @section("title", $titulo)

10. Resumen de tecnologías utilizadas

Bootstrap 5 (Componentes) Breadcrumb, tabla, botones (btn-dark, btn-outline-dark, btn-outline-secondary)
